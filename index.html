Day 3 in React

02/22/2022

Today is my third day learning React. I'll admit, my appreciation for React is growing everyday as I realise that what I'm learning speeds up my workflow by orders of magnitudes. I could make each of these blog posts from a single component since they're already nicely compartmentalized into divs with ids of "blog", with title and date tags. Easily done and very elementary in React.

I should have learned React a lot sooner. I'm currently learning react using this video which is a 10 hour comprehensive course. I'm doing an hour of this video everyday because I can't commit myself to watching more. I just get too distracted and I eventually click off to play video games or something.

I have been also working on some courses on Codeacademy as well. I opted to do the Learn React course to give myself some exercises and "course work" to get busy with. I'm not exactly enjoying this course, as it seems to use outdated/conflicting syntax from what I am acustomed to from the 10 hour video. For example, it advocates that one should always create a class which extends from react.Component with a render(){} function embeded in. I believe that function Component() (or for those who like arrow functions const Component = () => {}) works just as well. Needless to say, I'm still learning so there might be a very compelling reason why CodeAcademy advocates new learners to start using React in this manner.
UPDATE: So I found this brilliant technical article that compares the two methods of component creation I previously commented on. It should've been obvious when I first saw CodeAcademy's implementation of creating components by making classes which extend from React.Component. Extending them from the React.Component object in react is nice because it does what it says on the box: it's a class that houses the component. It's a factory that allows for getters and class functions which you can use when envoking render() instead of flanking your function Component(){} with other extreneous helper functions. Relevant constants stay within the scope of the class where they're relevant instead of floating around in global. I quite like this, but I still think I prefer creating my components with function Component(){}. For only spending 3 days with an average of maybe 2-3 hours on React, I'm really digging this framework. I just hope that the rest of the MERN stack is as enjoyable to learn.
